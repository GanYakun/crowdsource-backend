# 众包小程序后端 - 部署检查清单

## 部署前检查

### 1. 服务器环境
- [ ] 服务器操作系统：Linux（CentOS 7+/Ubuntu 18.04+）
- [ ] 内存：至少 2GB（推荐 4GB+）
- [ ] 磁盘空间：至少 10GB 可用
- [ ] Docker 已安装（20.10+）
- [ ] Docker Compose 已安装（2.0+）

验证命令：
```bash
docker --version
docker-compose --version
free -h
df -h
```

### 2. 网络配置
- [ ] 服务器可以访问外网（拉取 Docker 镜像）
- [ ] 阿里云安全组已开放 8080 端口
- [ ] （可选）域名已解析到服务器 IP
- [ ] （可选）SSL 证书已准备

### 3. 配置文件修改
- [ ] `docker-compose.yml` 中 `MYSQL_ROOT_PASSWORD` 已修改
- [ ] `docker-compose.yml` 中 `JWT_SECRET` 已修改（至少32位）
- [ ] 端口配置已确认（8080 未被占用）
- [ ] 时区配置正确（TZ=Asia/Shanghai）

### 4. 文件准备
- [ ] 项目文件已上传到服务器 `/opt/crowdsource-backend`
- [ ] `deploy.sh` 已添加执行权限（`chmod +x deploy.sh`）
- [ ] 数据库初始化脚本存在（`src/main/resources/db/schema.sql`）

## 部署步骤

### 第一步：上传项目
```bash
# 本地打包
.\package-for-deploy.ps1

# 上传到服务器
scp crowdsource-backend-deploy-*.tar.gz root@your-server-ip:/opt/

# 服务器解压
ssh root@your-server-ip
cd /opt
tar -xzf crowdsource-backend-deploy-*.tar.gz
cd crowdsource-backend
```

### 第二步：修改配置
```bash
# 编辑配置文件
vim docker-compose.yml

# 必须修改的配置：
# 1. MYSQL_ROOT_PASSWORD
# 2. JWT_SECRET
```

### 第三步：启动服务
```bash
# 添加执行权限
chmod +x deploy.sh

# 启动服务
./deploy.sh start

# 查看日志
./deploy.sh logs
```

### 第四步：验证部署
```bash
# 检查服务状态
./deploy.sh status

# 测试接口
curl http://localhost:8080/api/tags

# 从外网测试
curl http://your-server-ip:8080/api/tags
```

## 常见问题排查

### 问题1：容器无法启动
```bash
# 查看详细日志
docker-compose logs app

# 常见原因：
# - 端口被占用：netstat -tlnp | grep 8080
# - 数据库未就绪：等待 MySQL 健康检查通过
# - 配置错误：检查 docker-compose.yml 语法
```

### 问题2：数据库连接失败
```bash
# 检查 MySQL 状态
docker-compose ps mysql

# 查看 MySQL 日志
docker-compose logs mysql

# 测试连接
docker-compose exec mysql mysql -uroot -p
```

### 问题3：接口返回 401
```bash
# 检查 JWT 配置
# 确认 token 格式：Authorization: Bearer <token>
# 检查 token 是否过期
```

### 问题4：内存不足
```bash
# 查看内存占用
free -h
docker stats

# 调整 JVM 参数（编辑 docker-compose.yml）
JAVA_OPTS: -Xmx256m -Xms128m  # 降低内存占用
```

## 安全加固

### 1. 密码安全
- [ ] MySQL root 密码强度足够（至少16位，包含大小写字母、数字、特殊字符）

## 性能优化

### 1. JVM 调优
根据服务器内存调整：
- 1GB 内存：`-Xmx512m -Xms256m`
- 2GB 内存：`-Xmx1g -Xms512m`
- 4GB+ 内存：`-Xmx2g -Xms1g`

### 2. MySQL 调优
```yaml
# 在 docker-compose.yml 中添加
command:
  - --max_connections=200
  - --innodb_buffer_pool_size=256M
```

### 3. 连接池配置
编辑 `application.yml`：
```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
```
